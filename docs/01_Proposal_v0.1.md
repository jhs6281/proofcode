# ProofCode Initial Proposal v0.1

-   Date: 2026-07-11
-   Status: Initial Concept
-   Scope: 초기 구상 기록

## 1. 시작 아이디어

초기 아이디어는 사용자가 파일 또는 프로젝트 폴더를 입력하면 AI가 코드를
분석하고, 유사하거나 개선 가능한 로직을 제안한 뒤 실제 성능을 측정하여
결과를 전달하는 도구였다.

초기 흐름:

1.  파일 또는 폴더 입력
2.  현재 로직 분석
3.  AI가 대안 로직 제안
4.  대안 코드 실행
5.  속도, 메모리, CPU 비교
6.  결과 리포트

처음에는 Discord를 주요 출력 환경으로 고려했지만, 분석 결과, Diff,
그래프, 테스트 결과, 승인 기능을 표시하기에는 제한적이라고 판단했다.
Discord는 완료 알림과 짧은 요약에 사용하고, 주요 사용자 경험은 VS Code
Extension으로 제공하는 방향으로 변경했다.

## 2. 핵심 문제 발견

단순한 "AI 리팩토링 도구"는 기존 ChatGPT, Claude, Copilot 계열 도구와
차별성이 약하다.

따라서 핵심 가치를 다음과 같이 변경했다.

**AI가 코드를 고치는 도구 → AI가 제안한 변경을 검증하는 도구**

## 3. 안전한 변경 원칙

프로젝트 전체를 AI가 한 번에 수정하는 방식은 유지보수 위험이 크다.
따라서 다음 방식을 채택한다.

-   프로젝트 전체는 분석 가능하다.
-   병목과 개선 후보는 우선순위화한다.
-   변경 후보는 작은 단위로 생성한다.
-   AI는 직접 원본을 수정하지 않는다.
-   변경 후보는 Patch/Diff 형태로 관리한다.
-   검증은 격리된 작업 공간 또는 Git 기반 분리 환경에서 수행한다.
-   최종 적용은 개발자가 선택한다.

## 4. 검증 대상

초기 핵심 검증 항목:

-   기존 테스트 통과 여부
-   실행 시간
-   반복 측정 결과
-   평균, 중앙값, 최소/최대, 분산 또는 표준편차
-   메모리 사용량
-   CPU 사용량
-   기능 동등성 확인
-   변경 범위
-   복잡도 변화
-   유지보수 위험 신호

## 5. 주요 위험과 대응

### AI의 프로젝트 오해

전체 코드를 한 번에 모델에 전달하지 않는다. 프로젝트 구조를 먼저
인덱싱하고 관련 파일과 의존 관계를 선택적으로 Context에 제공한다. AI
설명은 Evidence가 아니며, 정적 분석과 실제 실행 결과를 분리한다.

### 성능 향상과 기능 손상

성능 측정 전에 기존 테스트를 실행한다. 테스트 실패 후보는 성능이 좋아도
추천 대상으로 승격하지 않는다.

### 부정확한 벤치마크

단일 실행 시간을 근거로 판단하지 않는다. 워밍업, 반복 실행, 이상치 처리
정책, 환경 정보 기록을 도입한다. 언어별 표준 벤치마크 도구를 Plugin으로
지원한다.

### 데이터 크기에 따른 결과 변화

하나의 데이터 크기만 사용하지 않는다. 가능한 경우 Small/Medium/Large
데이터 프로파일을 정의하고 동일 조건에서 원본과 후보를 비교한다. DB
성능은 애플리케이션 코드 벤치마크와 분리된 단계로 설계한다.

### 코드 유출

Provider 추상화를 적용한다. 외부 API뿐 아니라 로컬 모델 Provider를
지원할 수 있도록 설계한다. 비밀정보 탐지와 제외 규칙은 별도 보안 설계
항목으로 둔다.

### AI 환각

AI의 설명만으로 점수를 부여하지 않는다. 테스트, 벤치마크, 정적 분석 결과
등 검증 가능한 Evidence를 중심으로 판정한다.

### 속도만 좋아지고 유지보수성이 악화되는 문제

성능 외에도 메모리, CPU, 복잡도, 변경 범위, 테스트 결과를 함께 보여준다.
단일 종합점수는 상세 근거를 숨기지 않으며, 사용자가 각 지표를 직접
확인할 수 있어야 한다.

## 6. 주요 기능 아이디어

-   Evidence 기반 리뷰
-   Explain Why Not: AI 제안이 실패하거나 느려진 이유 설명
-   Replay: 분석 당시 모델, Prompt, Commit, 환경, 테스트와 벤치마크 조건
    저장
-   Multi-provider 비교
-   AI Court: 여러 AI 제안을 동일 조건에서 검증하고 결과 비교
-   Provider Plugin
-   Language Plugin
-   Benchmark Plugin
-   VS Code Extension
-   CLI
-   Discord 완료 알림

## 7. 초기 기술 방향

-   Core Engine: Python
-   VS Code Extension: TypeScript
-   UI: VS Code Extension 중심
-   AI 연결: Provider Interface
-   언어 지원: Plugin 구조
-   Benchmark: 언어별 Runner/Adapter 구조
-   Git: 안전한 변경 관리와 Diff 제공
-   Discord: 선택적 알림 채널

## 8. 초기 마일스톤

M1의 목표는 기능을 넓게 만드는 것이 아니라 첫 번째 "증명"을 완성하는
것이다.

1.  VS Code에서 분석 대상 선택
2.  Python Core Engine 호출
3.  하나의 AI Provider를 통한 개선 후보 생성
4.  격리된 환경에서 후보 검증
5.  테스트 및 벤치마크 결과 생성
6.  VS Code에서 Evidence 확인

M1에서는 자동 적용, 다중 AI 경쟁, DB 최적화, Discord, 보안 리뷰 전체
기능을 제외한다.
