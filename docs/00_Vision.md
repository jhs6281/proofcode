# ProofCode Vision

## Mission

ProofCode는 AI가 제안한 코드 변경을 개발자가 신뢰할 수 있도록, 객관적인
근거와 재현 가능한 검증 결과를 제공하는 오픈소스 개발자 플랫폼이다.

> AI는 코드를 변경하지 않는다.\
> AI는 근거를 제시하고 검증한다.\
> 최종 결정은 개발자가 한다.

추가 원칙은 다음과 같다.

-   모든 AI 제안은 재현 가능해야 한다.
-   AI의 의견과 측정 결과를 분리한다.
-   점수와 판정은 가능한 한 엔진의 실측 데이터에 기반한다.
-   원본 코드는 자동으로 직접 수정하지 않는다.
-   플랫폼은 특정 AI 모델보다 오래 살아야 한다.
-   프로젝트 전체를 분석하되, 변경 제안과 검증은 작은 단위로 수행한다.

## Problem

ChatGPT, Claude, Copilot 등의 AI 도구는 코드 생성, 설명, 리뷰, 리팩토링
제안을 빠르게 수행한다. 그러나 제안된 변경이 실제 환경에서 더 빠른지,
기존 동작을 유지하는지, 메모리나 CPU 비용을 과도하게 증가시키지 않는지,
유지보수성을 악화시키지 않는지는 별도의 검증이 필요하다.

ProofCode의 핵심 질문은 단순하다.

**"AI가 제안한 변경이 실제로 더 나은가?"**

## Product Philosophy

ProofCode는 자동 수정기가 아니다. 개발자의 판단을 대체하지 않는다.

플랫폼의 기본 흐름은 다음과 같다.

1.  프로젝트 구조와 실행 환경을 파악한다.
2.  병목 또는 개선 후보를 찾는다.
3.  AI Provider가 변경 후보와 근거를 제안한다.
4.  원본과 분리된 환경에서 후보를 검증한다.
5.  기존 테스트, 벤치마크, 자원 사용량과 위험 신호를 확인한다.
6.  Evidence를 개발자에게 보여준다.
7.  개발자가 적용, 보류, 거부를 결정한다.

## Identity

ProofCode는 AI 코드 생성기가 아니라 **Developer Trust Platform**을
지향한다.

Slogan:

**AI proposes. ProofCode verifies. Developers decide.**
